# 🌤️ Family Weather Dashboard

A private, AI-narrated daily weather broadcast for a family in Shulin/Sanxia/Banqiao (New Taipei City, Taiwan). Each day the app fetches live weather and air quality data, generates a natural-language narration via an LLM (Claude or Gemini), synthesises it to speech, and surfaces everything in a responsive dashboard.

---

## Features

| Feature | Details |
|---|---|
| **Live weather data** | Current conditions + 36-hour & 7-day forecasts via [CWA Open Data](https://opendata.cwa.gov.tw/) |
| **Air quality** | Real-time AQI + 3-day regional forecast via [MOENV](https://data.moenv.gov.tw/) |
| **AI narration** | Daily broadcast script generated by Claude (Sonnet/Haiku) or Gemini (Pro/Flash), with template fallback |
| **Text-to-Speech** | Audio rendered by Google Cloud TTS (`zh-TW-Wavenet-B` / `en-US-Neural2-D`) or Microsoft Edge TTS (offline fallback) |
| **Outdoor Score** | Rule-engine score (0–10) rating suitability for outdoor activity based on temp, humidity, AQI, UV, and visibility |
| **Health & Safety Alerts** | Early warnings for Cardiac risk (cold/wet swings) and Ménière's disease triggers (barometric drops) |
| **Lifestyle Advice** | Weather-driven meal suggestions, commuter hazard warnings (rain/wind), HVAC mode and wardrobing tips |
| **Language toggle** | English / Traditional Chinese (`zh-TW`) — narration, UI labels, and TTS voice all switch together |
| **PWA** | Service worker + web manifest for installable offline use |
| **Streaming refresh** | Pipeline progress streamed to the browser via NDJSON (Server-Sent Events) |
| **Narration cache** | 30-minute in-memory cache keyed by language + city + weather condition + time-of-day |
| **14-day regen cycle** | Meal & location database auto-regenerated by the LLM every two weeks |

## Architecture

```
┌──────────────────────────────────────────────────────────────┐
│                     Browser (PWA)                            │
│  dashboard.html  ←  app.js  +  style.css                     │
└──────────────────┬─────────────────────────────┬────────────┘
                   │ /api/refresh (NDJSON stream) │ /api/broadcast
                   ▼                             ▼
┌─────────────────────────────────────────────────────────────┐
│                    Flask (app.py)                            │
│  LOCAL / MODAL mode: runs pipeline directly                  │
│  CLOUD mode: proxies to Modal serverless endpoints           │
└──────┬──────────────┬──────────────┬────────────────────────┘
       │              │              │
  data/          narration/      history/
  fetch_cwa      llm_prompt_     conversation.json
  fetch_moenv    builder         (local file or GCS)
  weather_       claude_client
  processor      gemini_client
  outdoor_score  tts_client
                 llm_summarizer
```

### Run modes (`RUN_MODE` env var)

| Mode | Description |
|---|---|
| `LOCAL` | Flask dev server runs the full pipeline in-process. Reads/writes `local_data/`. |
| `CLOUD` | Flask acts as a thin proxy; the pipeline runs on [Modal](https://modal.com/) serverless workers which write to GCS. |
| `MODAL` | Used inside Modal workers themselves; reads/writes to a mounted Modal Volume at `/data`. |

## Setup

### Prerequisites

- Python 3.12+
- A [CWA Open Data](https://opendata.cwa.gov.tw/) API key
- A [MOENV](https://data.moenv.gov.tw/) API key
- An [Anthropic](https://console.anthropic.com/) API key (Claude)
- A [Google AI Studio](https://aistudio.google.com/) API key (Gemini)
- *(Optional)* A GCP service account with **Cloud TTS** and **Cloud Storage** permissions, for Google TTS and cloud history storage.

### 1. Clone & install

```powershell
git clone <repo-url>
cd family-weather
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 2. Configure environment

Copy `.env` and fill in your keys:

```dotenv
RUN_MODE=LOCAL

CWA_API_KEY=<your-key>
MOENV_API_KEY=<your-key>
GEMINI_API_KEY=<your-key>
ANTHROPIC_API_KEY=<your-key>

# Optional: override narration provider (CLAUDE | GEMINI | TEMPLATE)
NARRATION_PROVIDER=CLAUDE

# Optional: GCP for Cloud TTS & GCS history
# GCP_PROJECT_ID=...
# GCS_BUCKET_NAME=family-weather-dashboard
# GOOGLE_APPLICATION_CREDENTIALS=local_data/service-account.json
```

> For Google Cloud TTS, place your service account JSON at `local_data/service-account.json`. If absent, the app falls back to **Microsoft Edge TTS** (no credentials required).

### 3. Run locally

```powershell
.\run_local.ps1
# or:
$env:RUN_MODE = "LOCAL"; python app.py
```

Navigate to `http://localhost:8080`.

## Deployment

### Docker / Cloud Run

```powershell
# Build
docker build -t family-weather .

# Run locally via Docker
docker run -p 8080:8080 --env-file .env family-weather
```

Deploy the image to **Google Cloud Run**. Set all env vars as Cloud Run environment variables or via Secret Manager.

### Modal (serverless pipeline)

```powershell
# Deploy the pipeline workers
modal deploy backend/modal_app.py
```

Set your secrets in the Modal dashboard under the name `family-weather-secrets` (matching keys: `CWA_API_KEY`, `MOENV_API_KEY`, `ANTHROPIC_API_KEY`, `GEMINI_API_KEY`).

After deploying, set `MODAL_REFRESH_URL` and `MODAL_BROADCAST_URL` on your Cloud Run service to the generated Modal endpoint URLs.

## API Endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/` | Dashboard UI |
| `GET` | `/api/health` | Health check |
| `GET` | `/api/broadcast?date=YYYY-MM-DD` | Fetch broadcast data for a date |
| `POST` | `/api/refresh` | Run the full pipeline (streaming NDJSON) |
| `POST` | `/debug/log` | Browser log forwarding (LOCAL only) |

`POST /api/refresh` accepts a JSON body:

```json
{
  "date": "2026-02-25",
  "provider": "CLAUDE",
  "lang": "zh-TW"
}
```
