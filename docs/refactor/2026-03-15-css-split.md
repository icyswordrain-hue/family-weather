# Refactor: Split style.css into Multiple Plain CSS Files

**Date:** 2026-03-15
**Status:** Ready for execution
**Executor:** Sonnet (low context) — one batch per conversation

## Context

`web/static/style.css` is 5,151 lines. It's hard to navigate, and any change busts the entire browser cache. Split it into 7 plain CSS files (no build tools) for maintainability and granular caching.

## Execution Strategy

**Sequential batches, one Sonnet conversation per batch.** Each batch creates one new CSS file, deletes the corresponding lines from `style.css`, and commits. Do NOT run batches in parallel — they all modify the same source file.

**Token budget:** Each batch involves at most ~900 lines of CSS extraction. Well within Sonnet's context window when combined with these instructions.

## Target File Structure

```
web/static/
  fonts.css      ~1,630 lines  @font-face CJK unicode-range declarations
  tokens.css       ~125 lines  :root variables, reset, dark-mode vars, .hidden
  layout.css       ~750 lines  App shell grid, sidebar, right panel, compact header, scrollbar
  views.css        ~550 lines  Main panel, view containers, animations, lifestyle, narration
  dashboard.css    ~900 lines  Gauges, hero, timeline, forecast, alerts, 7-day grid
  player.css       ~700 lines  Player bar, player sheet, Ryokan scroll, controls
  chat.css         ~155 lines  Chat panel, messages, suggestion chips, input
```

## Link Tag Order (cascade matters)

```html
<link rel="stylesheet" href="/static/fonts.css?v=1" />
<link rel="stylesheet" href="/static/tokens.css?v=1" />
<link rel="stylesheet" href="/static/layout.css?v=1" />
<link rel="stylesheet" href="/static/views.css?v=1" />
<link rel="stylesheet" href="/static/dashboard.css?v=1" />
<link rel="stylesheet" href="/static/player.css?v=1" />
<link rel="stylesheet" href="/static/chat.css?v=1" />
```

`player.css` must load after `views.css` because it contains a `.main-panel { padding-bottom }` override for player-bar clearance.

## Mobile Overrides Strategy

The big `@media (max-width: 767px)` block (lines 3555–3849) gets **distributed** into each component's file — not kept as a separate `mobile.css`. Each file becomes self-contained: base rules + dark mode + responsive overrides.
